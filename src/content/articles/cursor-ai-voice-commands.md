Cursor, the AI-powered code editor, has shipped voice command support in version 0.45, allowing developers to write, edit, and navigate code using natural language voice input without touching the keyboard.

## How Voice Commands Work

Voice input is processed locally using a lightweight speech recognition model, with the transcribed text passed to Cursor's existing AI completion and chat systems. Latency in testing averaged under 400ms from speech to code action.

## Accessibility Impact

The feature has been received particularly positively by developers with repetitive strain injuries and other conditions that make sustained keyboard use difficult. Cursor has stated that accessibility was the primary motivation for the feature's development.

## Current Limitations

Voice commands work best for discrete instructions ("add a try-catch block around this function," "rename this variable to userConfig"). Complex multi-step refactors still require follow-up keyboard confirmation for accuracy.
